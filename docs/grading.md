# Grading

## Scoring Categories
| Category          | Weight | Description |
|------------------|--------|-------------|
| Functionality     | 40%    | Features work according to the User Story requirements; correctness of implementation |
| Code Quality      | 20%    | Clean, readable code following style guidelines; appropriate comments |
| Tests             | 15%    | Tests are written and pass without errors |
| CI/CD             | 10%    | Pull requests pass through CI without errors |
| Documentation     | 10%    | README, guides, and project documentation are updated for new changes |
| Demo / Presentation | 5%    | Results are presented; features are demonstrated during demo meetings |
