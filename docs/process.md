# Process Guidelines

## Definition of Ready (DoR)
- User Story is described with clear acceptance criteria
- Task is assigned to the appropriate intern
- Dependencies are identified

## Definition of Done (DoD)
- Code is written and passes local tests
- Unit/Integration tests are added and pass
- CI/CD build passes
- Documentation is updated (README, API docs, instructions)
- PR has three reviewers: two interns and one mentor. Reviewers must approve before merge.
- Acceptance tests pass successfully

## Pull Request Checklist
- [ ] User Story / Issue is linked
- [ ] Code passes local tests
- [ ] CI/CD build is passed
- [ ] Documentation is updated
- [ ] Reviewers are assigned
